# QMDスライド ビジュアルレビュー

## 目的

QMDファイル（revealjs）を読み取り、ビジュアルで理解を深められるスライドを見つけ出し、**コスト対効果の高い**提案を行う。テキストと関連するビジュアルの併用は学習効果を高める（マルチメディア効果）。ただし内容と無関係な装飾や、学生の能動的行動を阻害するビジュアルは提案しない。

## 2つの提案層

| 層 | 定義 | 例 | コスト |
|:---|:---|:---|:---|
| **Tier 1: 主役図** | スライドの中心的なビジュアル | インフォグラフィック、Rグラフ、写真 | 中〜高 |
| **Tier 2: 補助的ビジュアル** | テキストを強化するシンプルな図 | アイコン配置図、シンプルSVG、ドーナツチャート | 低 |

**Tier 2 こそがこのスキルの価値**。主役図（Tier 1）は QMD作成時に自然と計画される。見落とされがちな「低コストで効果のあるビジュアル」を提案するのがこのスキルの本領。

## verify-qmd との役割分担

- **verify-qmd**: INFOGRAPHICコメントが「存在するか」の機械的合否判定。レイアウト改善はverify-qmdの責務
- **visual-designer（本スキル）**: 「何が足りないか」「何を追加すべきか」の創造的提案

## 処理フロー

### Phase 0: 引数解析

- ファイルパス: 必須。指定がなければユーザーに確認
- `--set N`: セット番号で絞り込み（任意）
- `--slide N`: スライド番号（1-based）で絞り込み（任意）

### Phase 1: QMD構造解析

対象QMDファイルをReadで全文読み込み、`##` 見出しでスライド境界を分割する。
各スライドについて以下を抽出:

- CSSクラス（`.detail`, `.question-slide`, `.conclusion-first` 等）
- `<!-- INFOGRAPHIC: ... -->` コメントの有無と内容
- `::: {.notes}` 内の `📊 **インフォグラフィック依頼**` の有無
- `![](...)` 画像参照の有無
- `<!-- TODO: ... -->` の有無と内容 → **画像関連のTODOは Phase 3 で自動的に提案に変換する**
- テキスト量（概算行数）
- セット番号: 最初の `.question-slide` をセット1の開始とし、以降の `.question-slide` ごとにインクリメント。`.question-slide` 以前はPart 1（ガイダンス）

**整合性チェック**: `<!-- INFOGRAPHIC: -->` と `📊 インフォグラフィック依頼` の片方のみ存在する場合は警告。

### Phase 2: ビジュアル需要判定

スライドの教育的役割に基づいて判定する。

**提案対象の判定:**

| CSSクラス / 役割 | 提案対象か | 理由 |
|:---|:---|:---|
| `.question-slide` | 対象外 | 問いかけのみ。テキストの力で注意を引く |
| `.detail`（データあり） | **対象** | データの可視化で理解度が大幅向上 |
| `.detail`（概念説明） | **対象** | 因果図・プロセス図で抽象概念を具体化 |
| `.detail`（活用/共感パート） | **対象外** | 学生の能動的行動（投票・挙手・対話）を阻害するリスク。「余白」は設計意図 |
| `.conclusion-first` | 対象外 | まとめ。前スライドの図で既に視覚化済み |
| `.summary` | 対象外 | テキスト一覧で振り返り |
| `.roadmap` | **対象** | ジャーニーマップ・タイムライン図が教育効果大 |
| `.table-slide` | **対象** | 表の補完としてグラフがあると理解しやすい（Tier 2） |
| `.overview-3col` | **対象** | 3カラムのアイコン画像が教育効果に直結 |
| `.simulation` | 済（内在） | GIF/アニメ自体がビジュアル要素 |
| `.cta` / `.next-preview` / `.ending` | 対象外 | 行動喚起・次回予告・挨拶 |
| （クラスなし / 未知のクラス） | 要個別判断 | テキスト量とコンテンツから判定 |

**`.detail` の3分類ヒューリスティクス:**
- **データあり**: 数値・%・単位（ha, t, kg）が含まれる、または表がある
- **概念説明**: 因果関係の接続詞（「なぜなら」「その結果」「つまり」）、構造の説明、リスト3項目以上
- **活用/共感パート**: 投票マーク📱、挙手指示、「考えてみよう」「思い出してください」系の指示文、対話を促す問いかけ

**提案数の目安**: 対象候補スライド数（対象外を除いた枚数）の50-70%。母数にquestion-slide、conclusion-first等は含めない。

### Phase 3: 提案生成

以下の3つのソースから提案を生成する:

#### 3a. 画像TODOコメントからの自動変換

`<!-- TODO: ... 画像 ... -->` や `<!-- TODO: ... 写真 ... -->` を検出した場合、そのTODO内容に基づいてTier 1提案を自動生成する。TODOコメントは「作成者が既に必要性を認識している」証拠であり、スキルが改めて必要性を判定する必要はない。

#### 3b. 対象スライドへの新規提案

Phase 2 で「対象」と判定され、かつ既存のINFOGRAPHICコメント・画像・TODOがないスライドに対して提案を生成する。

**コスト対効果で判定する**（二値の「必要/不要」ではない）:

| 改善度＼コスト | 低（R5行/シンプルSVG/CC写真） | 中（Rグラフ/SVG） | 高（複雑SVG） |
|:---|:---|:---|:---|
| **大**（数値3つ以上/因果2段階以上） | **必須** | **推奨** | 推奨 |
| **中**（比較2つ以上/手順3ステップ以上/リスト4項目以上） | **推奨** | 任意 | 低優先 |
| **小**（上記に該当しない） | 任意 | 低優先 | 提案しない |

**改善度のヒューリスティクス:**
- **大**: 数値データが3つ以上含まれる / 因果関係が2段階以上ある / 時系列データがある
- **中**: 比較対象が2つ以上 / 手順・プロセスが3ステップ以上 / リストが4項目以上でアイコン化すると記憶に残る
- **小**: 上記に該当しない概念説明、短いテキスト

**阻害リスクチェック**: 学生の能動的行動を促すスライド（投票・挙手・対話・思考）では、ビジュアルが行動を阻害しないか確認する。阻害リスクが高い場合は提案しない。

#### 3c. 整合性チェック

既にINFOGRAPHICコメントがあるスライドについて、`<!-- INFOGRAPHIC: -->` と `📊 インフォグラフィック依頼` の両方が揃っているかチェック。

**Tier の判定**:
- スライドの中心コンテンツを置き換える or 追加する → Tier 1
- 既存テキストの横や上に補助的に添える → Tier 2

図の型の選択は [references/visual-types.md](references/visual-types.md) を参照。参照ファイルがない場合は以下の最小限の型一覧で判断:

| 型 | 適用場面 | 実装手段 |
|:---|:---|:---|
| 因果フロー図 | 原因→結果の連鎖 | SVG (`/svg-precision`) |
| 比較図 | A vs B、before/after | 数値→R / 概念→SVG |
| プロセス図 | 手順・段階 | SVG |
| 構造図 | 部品・部位・階層 | SVG |
| 棒グラフ / 折れ線 / 円グラフ / ドーナツ | 数値データの可視化 | R (`ggplot2`) |
| ジャーニーマップ | 時系列・ロードマップ | SVG |
| アイコン配置図 | リスト項目の視覚化 | SVG（シンプル） |
| 教育イラスト | 概念図・情景 | nanobanana pro (lecture-framework) |
| 写真 | 実物・装置・現場 | CC画像取得（要著作権確認） |

**セット横断の多様性チェック**: Tier 1 提案が5件以上の場合のみ実行。Tier 2 は多様性チェックの対象外。

### Phase 4: 自己検証

各提案に以下のチェックを適用する:

| # | チェック項目 | 判定基準 |
|:---|:---|:---|
| V1 | **コスト対効果**: 改善度に対してコストは見合うか？ | Phase 3 のマトリクス参照 |
| V2 | **教育的位置**: 4ステップの何に対応するか？ | 全体像 / ゴール / アクション / 具体例 / 不明確 |
| V3 | **阻害リスク**: 学生の能動的行動を妨げないか？ | なし / あり（ありなら撤回） |
| V4 | **レイアウト実現性**: revealjsで実装可能か？ | 可能 / 要調整 |
| V5 | **Tier**: 主役図か補助的ビジュアルか？ | Tier 1 / Tier 2 |

- V2が「不明確」→ 低優先度として末尾に付記
- V3が「あり」→ 撤回

### Phase 5: 出力

以下の構造でユーザーに報告する:

```markdown
## ビジュアルレビュー: [ファイル名]

### サマリー
| 区分 | スライド数 |
|:---|---:|
| ビジュアル済み（INFOGRAPHIC/画像あり） | X |
| 対象外（テキストで十分/活用パート等） | X |
| **Tier 1 提案（主役図）** | **X** |
| **Tier 2 提案（補助的ビジュアル）** | **X** |
| 整合性チェック指摘 | X |

### Tier 1 提案（主役図）

#### 提案N: 「スライドタイトル」（セットN, .CSSクラス）

- **現状**: （テキストのみ / TODO画像あり / 等）
- **提案**: （ビジュアルの内容）
- **型**: （因果フロー / 比較図 / Rグラフ / 写真 / 等）
- **理由**: （なぜこの型が適切か、1-2文）
- **制作方法**: （/svg-precision / R ggplot2 / nanobanana / CC画像取得）
- **レイアウト**: （.main-visual / .columns 50:50 / 等）
- **ファイル名**: （NN_NN_説明.拡張子）
- **自己検証**: V1=推奨 V2=具体例 V3=なし V4=可能 V5=Tier1

**INFOGRAPHICコメント案（`##` 見出し直後に追記）:**
```qmd
<!-- INFOGRAPHIC: 型 — 内容の要約 -->
```

**スピーカーノート追記案:**
```qmd
📊 **インフォグラフィック依頼**:
- 種類: ...
- 内容: ...
- 強調: ...
- レイアウト: ...
- 制作方法: ...
```

### Tier 2 提案（補助的ビジュアル）

（同様のフォーマット。Tier 2 はコメント案を簡略化してもよい）

### 整合性チェック指摘
（既存INFOGRAPHICへの指摘、片方のみ存在の警告等）

### 次のアクション
- [ ] 提案Nを承認
  - `<!-- INFOGRAPHIC: ... -->` を `##` 見出し直後に追記
  - スピーカーノートに📊インフォグラフィック依頼を追記
  - 指定の制作方法で図を制作
```

## 注意事項

- このスキルはビジュアルの「提案」のみ行い、QMDファイルの編集は行わない
- 提案の承認後、メインエージェントがQMDに追記する
- 図の「制作」は `/svg-precision` やRスクリプトなど別のツール/スキルで行う
- **活用/共感パートには原則ビジュアルを提案しない** — 学生の能動的行動の「余白」は設計意図
