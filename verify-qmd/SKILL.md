# QMD完成チェック

## 目的

QMDファイルをプランや合意事項と照合し、構造・コンテンツ・品質の3層で検証する。批判的レビューの前段として、機械的に検出可能な問題を先に潰す。

## チェック項目

### 層1: 構造チェック（自動検出可能）

| # | チェック項目 | 方法 | 合格基準 |
|:-:|:---|:---|:---|
| S1 | スライド枚数 | `##` 見出しをカウント（ネストされたcallout内の見出しは除外） | プランの枚数と一致 |
| S2 | CSSクラス分布 | 各 `{.クラス名}` をカウント | `.detail` 以外も使われている |
| S3 | CSSクラス連続 | 同一クラスが3枚以上連続する箇所を検出 | 3連続以上がない（警告レベル） |
| S4 | スピーカーノート | 全スライドに `::: {.notes}` が存在するか | 全スライドに存在 |
| S5 | speaker-trigger | 全スライドに `::: {.speaker-trigger}` が存在するか | 全スライドに存在 |
| S6 | TODOコメント | `<!-- TODO:` の一覧化 | 各TODOの内容が具体的 |
| S7 | 画像参照 | `![](` の参照先ファイルが存在するか | 全画像が存在（または明示的TODO） |
| S8 | レンダリング | `quarto render` の成功 | エラーなし |
| S9 | 画像配置 | `![](` が `::: {.column` の中にあるか | `.columns` 外に単独画像がない（`.roadmap-icon`, `.inline-icon`, `.session-icon` は除外） |
| S10 | alt text | `![xxx](` の xxx が空か | 全画像のalt textが空（キャプション自動生成を防止） |
| S11 | カラー絵文字 | スライド本文（notes/speaker-trigger外）にUnicode絵文字がないか | スライド表示部分にカラー絵文字がない。`img/icons/*.svg` の `.inline-icon` で置換済み |
| S12 | 配色準拠 | 単純なSVG・グラフのINFOGRAPHICコメント内にカラー名がないか（詳しいイラスト・写真はカラー許可） | 単純SVG・グラフはグレースケール指示のみ。詳しいイラスト（nanobanana等）はカラー可 |
| S13 | セッションタイトル | `.session-title` スライドが1枚以上あるか。SVGアイコン + `.session-icon` を使用しているか | 形式が正しい（数の検証はC7に委任） |
| S14 | ロードマップ | `.roadmap` スライドが1枚以上あるか | 大A（`.key-phrase`）の直後に存在。小Q×4の予告として機能 |
| S15 | QARテンプレート順序 | 各セット内で `.session-title` → `.key-phrase` → Rスライド群 → `.set-summary` の順序か | Q→A→R→まとめの順序が全セットで守られている |
| S16 | セットまとめ | `.set-summary` スライドがセット数と同数あるか | 4セットなら4枚 |
| S17 | キーフレーズ出現回数 | `.key-phrase` クラスの出現回数 | 大A×1 + 小A×4 = 5回 |
| S18 | 旧クラス名残存 | `conclusion-first`, `conclusion-box`, `conclusion-visual` が残っていないか | 残存なし（移行完了確認） |
| S19 | 全体まとめ | `.summary` スライドが1枚あるか | 大Q+大A再提示として存在 |
| S20 | YAML直後の空白スライド | YAML閉じ `---` と最初の `##` の間にHTMLコメント・空行等のコンテンツがないか | `---` の直後が `##` であること（HTMLコメントも空スライドを生成する） |
| S21 | SVG内容品質 | `img/` 配下の全SVGが (a) viewBoxを持つか (b) stroke/fill色がグレースケール系か（アイコンは `#666`、インフォグラフィックは `#333`/`#999`/`#bbb`/`#ddd`/`#e8e8e8`/白/黒） (c) カラー絵文字テキスト要素（`☀️`等）を含んでいないか。S7は「ファイルが存在するか」、S11は「QMD本文の絵文字」、S21は「SVGファイルの中身が規約に沿っているか」 | SVG内容が品質基準を満たす |
| S22 | CSSスコープ整合性 | QMD内のスライドCSSクラス（`.detail`, `.overview-3col`等）と、スライド内の画像クラス（`.main-visual`, `.col-icon`等）の組み合わせが `style/base.scss` に定義されているか。例: `.overview-3col` 内で `.main-visual` を使う場合、`.reveal .overview-3col img.main-visual` のルールがあるか確認。未定義だと画像サイズが制御されない | 全組み合わせが定義済み |
| S23 | key-phrase画像クラス | `.key-phrase` スライド内の画像に `.key-phrase-visual`（写真: cover切り抜き）または `.key-phrase-illustration`（イラスト: contain全体表示）クラスが付いているか（`.main-visual` ではなく）。key-phraseスライドは固有のCSSコンテナ（65vh固定）を持つため、クラスの誤りはレイアウト崩れにつながる | `.key-phrase` 内の画像が全て `.key-phrase-visual` or `.key-phrase-illustration` |
| S24 | key-message文字数 | `.key-message` を含むスライドで、カラム幅に応じた文字数上限を超えていないか確認。100%幅（カラム外）→30字、50%カラム→20字、40-45%カラム→15字。Markdown記法（`**`等）は除去してカウント。同一カラム内の箇条書き4項目以上 + key-messageの組み合わせは致命的。参照: `reference/講義テンプレート設計.md` のkey-message文字数ガイドライン | 文字数が閾値以内（警告） |
| S25 | 出典名正式名称 | `.source` クラスおよびスピーカーノート内の出典文字列を抽出し、対応する `reference/2026_データ調査_*.md` の出典名と照合。略称→正式名称の不一致があれば警告 + 正式名称を提示。例: 「園芸用施設設置状況」→正しくは「園芸用ガラス室等の設置状況」 | 出典名が正式名称と一致（警告） |
| S26 | カラム内テキスト量 | `.columns` 内の各 `.column` のテキスト行数をカウント（箇条書き + key-message + source等の総量）。カラム幅と画像の有無から利用可能な縦スペースを推定し、超過時に警告。画像あり50%カラムでテキスト6行以上、画像あり40%カラムで4行以上を閾値とする | テキスト量が推定スペース以内（警告） |
| S27 | 2画像並置警告 | `.columns` 内に2つの `.column` がそれぞれ画像を含むスライドを検出。画像の下にkey-messageやsource等のテキスト要素がある場合、画像の `max-height` 制限（`max-height:45vh` 等）が明示されていなければ通知 | `max-height` が指定済み（情報） |
| S28 | key-phraseカラム画像アスペクト比 | `.key-phrase` スライドの `.columns` 内にある `.key-phrase-illustration` 画像のアスペクト比を `identify` コマンドで確認。幅/高さ比 > 2.0（横長すぎ）の場合、カラム内でcontain表示が小さくなる旨を警告。縦長 or 正方形の画像を推奨、または `.key-phrase-visual`（cover切り抜き）への変更を提案 | 幅/高さ比 ≤ 2.0（警告） |
| S29 | key-phrase source配置 | `.key-phrase` スライドの `.columns` 直後に `.source` がある場合、画像の表示領域を圧迫しやすい。sourceをスピーカーノート内に移動することを提案 | sourceがcolumns外にない（情報） |
| S30 | SVG width/height属性 | QMDで参照されている `.svg` ファイルの `<svg>` タグに `width` と `height` 属性があるか確認。未指定だと `<img>` 表示時にブラウザが固有寸法を算出できず高さ0になる | width/height属性あり（致命的） |
| S31 | SVGフォントパス変換 | QMDで参照されている `.svg` ファイルに `<text>` 要素が残存していないか確認。`<img>` 読み込みではWebフォントが適用されないため、テキストはInkscapeでパス変換が必要。`img/svg-src/` に編集用テキスト版があるか確認 | `<text>` 要素が0個（警告） |

### 層2: コンテンツチェック（半自動）

| # | チェック項目 | 方法 | 合格基準 |
|:-:|:---|:---|:---|
| C1 | キーフレーズ接触 | `.key-phrase-box` 内のキーフレーズ文字列の出現回数と位置 | 大A: 冒頭+全体まとめ、小A: 各セット冒頭+set-summary |
| C2 | セットゴール | 各Qスライドのノートに「セットゴール」が明記されているか | 全セットに存在 |
| C3 | ビジュアルプラン | 画像を含むスライド（`![](` がある）の対応するnotesに `📷 **ビジュアルプラン**` または `📊 **ビジュアルプラン**` が存在するか確認。存在する場合、必須フィールド「種別」「意図」があるか確認。プランなしの画像スライドは警告（Phase 2以降で追加された場合は許容）。旧記法 `📊 **インフォグラフィック依頼**` も互換として認識する | 画像スライドにビジュアルプランが対応（警告） |
| C4 | データ数値の突合 | リファレンスファイルの数値とスライド上の数値を比較。加えて、`reference/2026_データ調査_*.md` 内の⚠️マーク付き注記を明示的に抽出し、QMDでその注記が対処されているか確認（例: 「露地を含む」という注記がQMD内で明示されているか） | 一致（丸めは許容、矛盾は不可）。⚠️注記の対処漏れなし |
| C5 | 出典表記 | `.source` クラスまたはノート内出典の存在 | データを含むスライドに出典がある |
| C6 | 投票スライド | `smartphone.svg` の `.inline-icon` がスライドタイトルに存在するか。該当スライド内に `QR_` を含む画像があるか。対応するspeaker-triggerに `viewanalytics` リンクがあるか | 投票スライドにQR画像と結果リンクが揃っている |
| C7 | セッションタイトル数 | `.session-title` の枚数がセット数+ガイダンス有無と一致するか | 4セット+ガイダンス=5枚（第1回）、4セット=4枚（第2回以降） |

### 層3: 品質チェック（判断を伴う）

| # | チェック項目 | 方法 | 合格基準 |
|:-:|:---|:---|:---|
| Q1 | 4ステップ入れ子 | Part 1のノートに4ステップラベルがあるか | 全体像・ゴール・アクション・具体例が明示 |
| Q2 | 感情アーク | セット構成が計画された感情順序に沿っているか | 計画通り |
| Q3 | ノートの実用性 | スピーカーノートが講師として実際に話せるレベルか | 発話例が含まれている |
| Q4 | 決定事項照合 | `/capture-decisions` のチェックリストと照合 | 全項目が実装済み |

## 実行手順

1. 対象QMDファイルを読む
2. プランファイルがあれば読む（検証基準の取得）
3. `/capture-decisions` の出力があれば読む（合意事項の取得）
4. リファレンスファイルがあれば読む（データ突合用）
5. 層1→層2→層3の順にチェック実行
6. 結果をサマリーテーブルで出力
7. 問題があれば致命的/警告/情報の3段階で分類


## `/visual-check` との連携

verify-qmdは**ソースレベル**の構造チェック、`/visual-check`は**レンダリング後**の視覚チェック。推奨実行順序:

1. `/verify-qmd` — QMDの構造・コンテンツ・品質チェック
2. `quarto render` — HTMLを生成
3. `/visual-check` — 実際の表示を自動スクリーンショット・DOM測定で検証

S22はvisual-checkのVC1-VC5問題を**事前に予防**するためのもの。S24-S27はlec01で頻発したはみ出しパターンの事前予防。

## `/critical-review` との連携

verify-qmdは**機械的チェック**、`/critical-review`は**意味的レビュー**。verify-qmdを先に実行して機械的に検出可能な問題を潰してから、critical-reviewで意味的レビューに集中する。

- verify-qmdのC4が「数値一致」と判定した場合でも、critical-reviewのCR1は「その解釈は正しいか」を問う
- verify-qmdのC3でビジュアルプランの存在を確認し、critical-reviewのフルレビューでプランと実装の乖離を検出する

## 拡張ポイント

このスキルはQMD修正を重ねるたびにチェック項目を追記して育てる。以下の領域が将来の追加候補:

- アクセシビリティ（alt属性の有無、コントラスト比）
- 配布版との整合性（ノートが配布版で適切に表示されるか）
- 時間配分の妥当性（スライド枚数 vs 割当時間）
- 前回・次回講義との接続チェック（次回予告の整合性）
