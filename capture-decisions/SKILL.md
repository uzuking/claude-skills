# ブレスト決定事項キャプチャ

## 目的

会話中に散在する合意・決定事項を漏れなく抽出し、構造化されたチェックリストとして保存する。実装フェーズでの「合意したのに忘れた」問題を防ぐ。

## 処理フロー

### 1. 会話スキャン

現在の会話コンテキストを最初から最後まで精査し、以下を全て拾い上げる:

- ユーザーが明示的に決めたこと（「〜にしよう」「〜でいこう」）
- 議論の結果として合意したこと（「それでいい」「同意」「了解」）
- 批判的レビューの結果として採用された修正
- 暗黙の合意（提案に対して異議なく次に進んだもの）
- 否定された選択肢（「〜はやらない」「〜は入れない」）— これも記録する

### 2. 分類

抽出した決定事項を以下の4カテゴリに分類:

| カテゴリ | 内容例 |
|:---|:---|
| **構造的決定** | レイアウト、順序、時間配分、セット構成 |
| **コンテンツ決定** | データ、テキスト、画像、キーフレーズの配置 |
| **制作方法の決定** | ツール、フォーマット、仕様、備考の書き方 |
| **品質要件** | チェック基準、レビュー条件、検証項目 |

### 3. 出力

以下のテンプレートでMarkdownリストを生成:

```markdown
## ブレスト決定事項（YYYY-MM-DD）

### 構造的決定
- [ ] [決定事項1]
- [ ] [決定事項2]

### コンテンツ決定
- [ ] [決定事項3]

### 制作方法の決定
- [ ] [決定事項4]

### 品質要件
- [ ] [決定事項5]

### 明示的に却下された案
- ~~[却下案1]~~ — 理由: [なぜ却下されたか]
```

### 4. 保存

保存先はユーザーに確認。デフォルトの候補:
- プランファイルの末尾に追記（プラン作成前の場合）
- セッション記録に含める（セッション終了時の場合）
- 独立ファイルとして保存（ユーザー指定の場合）

## 注意事項

- **漏れを恐れる**: 「たぶん決まったこと」も含める。過剰に拾って後で削る方が、拾い漏れるより安全
- **暗黙の否定も記録**: 「〜はやらない」と決めたことも重要。実装時に「あれ、これやるんだっけ？」を防ぐ
- **数値は具体的に**: 「キーフレーズは4回」のように、検証可能な形で記録する
- **後から追記可能**: チェックリストは実装中に項目を追加してよい
