# 批判的レビュー

## 目的

verify-qmdが検出できない**意味的問題**を、提案者のスタンスで指摘する。「間違いです」ではなく「ここは大丈夫ですか？」と問いかけ、修正提案を必ず添える。

## verify-qmdとの役割分担

| | verify-qmd | critical-review |
|:---|:---|:---|
| 判断の性質 | 機械的（有/無、数/数の照合） | 意味的（「本当にそう言えるか」） |
| 入力 | QMDファイル + プラン | QMDファイル + 作業の文脈 + referenceドキュメント |
| 出力 | 合否テーブル | 問題指摘 + 修正提案（対話型） |
| スタンス | ゲートキーパー（基準に照合） | 提案者（「大丈夫ですか？」と問いかけ） |

verify-qmdが「合格」としたデータも、critical-reviewは「その解釈は正しいか」を問う。両者は補完関係であり、どちらかで代替できない。

## チェック観点（CR1-CR5）

### CR1: 事実正確性（Factual Accuracy）

スライド上の事実・データが正確か。

確認事項:
- 統計データの正式名称がリファレンスファイルと一致するか（略称の混在に注意）
- 数値とその文脈が正確か（「4割」が何の4割か、露地含むか施設単独か）
- 著者名・出典の表記が正確か
- Rスクリプトのcaption・ラベルに含まれる出典名

lec01での検出実績: 「園芸用施設設置状況」→正しくは「園芸用ガラス室等の設置状況」、「4割」→「38%（露地含む）」

### CR2: 論理的整合性（Logical Coherence）

スライド内およびスライド間の論理が通っているか。

確認事項:
- key-messageがそのスライドのテーマと論理的に対応するか
- 連続するスライドのkey-messageが論理的に接続するか（積み上げ・対比・展開）
- 小Qに対して小Aが「答え」になっているか（定義型で答えていないか、具体例型が適切か）
- 因果関係の主張に根拠があるか
- セット内の流れが小Q→探究→小Aの論理構造を持つか

lec01での検出実績: 植物工場→砂漠→宇宙のkey-messageが1枚ずつズレ、「自分に関係ある？」に定義で答えていた

### CR3: 教育的適切性（Pedagogical Appropriateness）

初学者にとって理解・受容しやすい表現・構成か。

確認事項:
- 専門用語の初出時に説明または具体例があるか
- 学生の既有知識から無理なく到達できる論理展開か
- 共感パート・活用パートが学生の能動的行動を促すか（一方的に答えを与えていないか）
- 比喩・表現が意図しない連想を生まないか
- インタラクション設計が対象学生に適切か（岩手大: 挙手ではなくスマホ匿名投票）

lec01での検出実績: 「巨大技術」がダム的連想を生む問題、施設園芸の定義が暗黙的だった

### CR4: 一次ソースとの整合（Source Alignment）

referenceドキュメントの情報がQMDに正しく反映されているか。

確認事項:
- `reference/2026_データ調査_*.md` 内の⚠️マーク付き注記がQMDで対処されているか
- 基盤文書（`reference/2026_講義全体像.md`）のキーフレーズ・セット構成と一致しているか
- データ調査で「区別せよ」と警告されている概念（例: 園芸農業 vs 施設園芸）がQMDで混同されていないか
- スピーカースタイルガイドの言い回し指針に沿っているか

lec01での検出実績: データ調査に「露地を含む38.5%」の警告があったのにQMD未反映

### CR5: 成果物間の一貫性（Cross-Artifact Consistency）

QMD内の複数箇所、および関連ファイル間で矛盾がないか。

確認事項:
- 同じ概念を異なる表現で言及していないか（用語の統一）
- CSSクラス名の旧→新移行が完了しているか
- スピーカーノートとスライド本文で矛盾する情報がないか
- 同一セッションで修正した複数ファイル間（QMD、Rスクリプト、reference）の整合

lec01での検出実績: `.conclusion-first`→`.key-phrase` 移行漏れ3箇所、「栽培施設」と「施設園芸」の混在

## 実行モード

### モード1: ステップレビュー（デフォルト）

- **タイミング**: 1つの作業ステップが完了した直後
- **対象**: そのステップで変更した範囲のみ
- **適用観点**: CR1-CR5のうち変更内容に関連するもののみ
- **所要**: 短い（変更範囲に集中）

引数なしまたは `/critical-review` で実行。直前の変更内容から対象範囲を自動判定。

### モード2: セットレビュー

- **タイミング**: 1セット（小QAR単位）の全スライドが揃った段階
- **対象**: 指定セットの全スライド群
- **適用観点**: CR1-CR5を全て適用。特にCR2（key-messageの論理的接続）に注力
- **所要**: 中程度

`/critical-review --set N` で実行。Nはセット番号（1-4）。

### モード3: フルレビュー

- **タイミング**: 講義全体が完成に近い段階（Phase 4相当）
- **対象**: QMD全体 + 関連reference全て
- **適用観点**: CR1-CR5を網羅的に適用
- **追加観点**:
  - セット間の整合性（4セットが大Aの分解として機能しているか）
  - 感情アークの流れ（計画された感情順序の実現度）
  - **ビジュアルプランとの乖離**: notesのビジュアルプラン（種別・意図）と実際のビジュアルを照合。意図的でない乖離を検出
- **所要**: 長い（全体を精査）

`/critical-review --full` で実行。

## 実行手順

1. **対象の特定**: モードに応じた範囲を決定
2. **文脈の読み込み**: 作業の目的（何を修正したか、なぜ修正したか）を把握
3. **referenceの参照**: 関連する `reference/` ドキュメントを読む（特にデータ調査の⚠️注記）
4. **CR1-CR5を順に確認**: 各観点の確認事項に沿ってチェック
5. **結果を出力**: 問題があれば指摘 + 修正提案。問題がない観点も「確認済み」として報告
6. **ユーザーとの対話**: 指摘について議論し、修正方針を合意

## 出力フォーマット

```markdown
## 批判的レビュー: [対象の説明]
**モード**: ステップ / セット / フル

### 検出された問題
| # | 観点 | 重要度 | スライド | 問題 | 修正提案 |
|:-:|:---|:---|:---|:---|:---|
| 1 | CR1 | 致命的 | #14 | ... | ... |

### 確認済み（問題なし）
- CR1 事実正確性: [確認した範囲と結果]
- CR2 論理的整合性: [確認した範囲と結果]
- ...

### 今回の範囲外（次回確認推奨）
- [フルレビューでのみ確認可能な項目等]
```

## 重要な原則

- **提案者であり、ゲートキーパーではない**: ユーザーが「これでよい」と判断した場合はそれを尊重する
- **段階的適用が最も効果的**: 一括フルレビューより、各ステップ後のレビューの積み重ねが問題を早期検出する
- **省略するとユーザーが発見するまで問題が残る**: 面倒でも各ステップ後に実行する価値がある
- **修正提案を必ず添える**: 問題指摘だけで終わらない。「ここが問題です」→「こう修正してはどうですか」
- **verify-qmdを先に実行する**: 機械的に検出可能な問題を先に潰してから、意味的レビューに集中する
