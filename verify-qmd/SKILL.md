# QMD完成チェック

## 目的

QMDファイルをプランや合意事項と照合し、構造・コンテンツ・品質の3層で検証する。批判的レビューの前段として、機械的に検出可能な問題を先に潰す。

## チェック項目

### 層1: 構造チェック（自動検出可能）

| # | チェック項目 | 方法 | 合格基準 |
|:-:|:---|:---|:---|
| S1 | スライド枚数 | `##` 見出しをカウント（ネストされたcallout内の見出しは除外） | プランの枚数と一致 |
| S2 | CSSクラス分布 | 各 `{.クラス名}` をカウント | `.detail` 以外も使われている |
| S3 | CSSクラス連続 | 同一クラスが3枚以上連続する箇所を検出 | 3連続以上がない（警告レベル） |
| S4 | スピーカーノート | 全スライドに `::: {.notes}` が存在するか | 全スライドに存在 |
| S5 | speaker-trigger | 全スライドに `::: {.speaker-trigger}` が存在するか | 全スライドに存在 |
| S6 | TODOコメント | `<!-- TODO:` の一覧化 | 各TODOの内容が具体的 |
| S7 | 画像参照 | `![](` の参照先ファイルが存在するか | 全画像が存在（または明示的TODO） |
| S8 | レンダリング | `quarto render` の成功 | エラーなし |

### 層2: コンテンツチェック（半自動）

| # | チェック項目 | 方法 | 合格基準 |
|:-:|:---|:---|:---|
| C1 | キーフレーズ接触 | キーフレーズ文字列の出現回数と位置 | プラン指定の回数・位置と一致 |
| C2 | セットゴール | 各Qスライドのノートに「セットゴール」が明記されているか | 全セットに存在 |
| C3 | インフォグラフィック備考 | ノート内に「📊 **インフォグラフィック依頼**」が存在するか | プラン指定の箇所に存在 |
| C4 | データ数値の突合 | リファレンスファイルの数値とスライド上の数値を比較 | 一致（丸めは許容、矛盾は不可） |
| C5 | 出典表記 | `.source` クラスまたはノート内出典の存在 | データを含むスライドに出典がある |
| C6 | 投票マーク | 📱 の出現位置 | プラン指定の箇所に存在 |

### 層3: 品質チェック（判断を伴う）

| # | チェック項目 | 方法 | 合格基準 |
|:-:|:---|:---|:---|
| Q1 | 4ステップ入れ子 | Part 1のノートに4ステップラベルがあるか | 全体像・ゴール・アクション・具体例が明示 |
| Q2 | 感情アーク | セット構成が計画された感情順序に沿っているか | 計画通り |
| Q3 | ノートの実用性 | スピーカーノートが講師として実際に話せるレベルか | 発話例が含まれている |
| Q4 | 決定事項照合 | `/capture-decisions` のチェックリストと照合 | 全項目が実装済み |

## 実行手順

1. 対象QMDファイルを読む
2. プランファイルがあれば読む（検証基準の取得）
3. `/capture-decisions` の出力があれば読む（合意事項の取得）
4. リファレンスファイルがあれば読む（データ突合用）
5. 層1→層2→層3の順にチェック実行
6. 結果をサマリーテーブルで出力
7. 問題があれば致命的/警告/情報の3段階で分類

## 出力フォーマット

```markdown
## QMD検証レポート: [ファイル名]

### サマリー
| 層 | 合格 | 警告 | 問題 |
|:---|---:|---:|---:|
| 構造 | X | X | X |
| コンテンツ | X | X | X |
| 品質 | X | X | X |

### 問題一覧
| # | 層 | 重要度 | 内容 | 修正案 |
|:-:|:---|:---|:---|:---|

### TODO一覧（残作業）
- [ ] ...
```

## 拡張ポイント

このスキルはQMD修正を重ねるたびにチェック項目を追記して育てる。以下の領域が将来の追加候補:

- アクセシビリティ（alt属性の有無、コントラスト比）
- 配布版との整合性（ノートが配布版で適切に表示されるか）
- 時間配分の妥当性（スライド枚数 vs 割当時間）
- 前回・次回講義との接続チェック（次回予告の整合性）
